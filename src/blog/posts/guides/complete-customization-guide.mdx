## Configurable Call-to-Actions

AI CMS includes a powerful system for customizing call-to-action sections throughout your site. You can configure CTAs via environment variables without touching any code.

### Available CTA Locations

The system includes CTAs in these strategic locations:

1. **Blog Post CTA**: Appears after each blog post to drive engagement
2. **Footer CTAs**: Two promotional sections above the footer
3. **Custom CTAs**: Can be added to any page using the `<CTASection>` component

### Environment Configuration

Each CTA can be configured using environment variables. Here's the pattern:

```env
# CTA Configuration Pattern
VITE_CTA_{NAME}_{PROPERTY}="value"
```

### Blog Post CTA Example

Configure the CTA that appears after blog posts:

```env
# Blog Post CTA Configuration
VITE_CTA_BLOG_POST_HEADING="Ready to Build Your Own CMS?"
VITE_CTA_BLOG_POST_SUBHEADING="Start creating amazing content with AI CMS. Clone the repository and get started in minutes."
VITE_CTA_BLOG_POST_BUTTON_TEXT="Get Started"
VITE_CTA_BLOG_POST_BUTTON_URL="https://github.com/yourusername/ai-cms"
VITE_CTA_BLOG_POST_ENABLED="true"
```

### Footer CTAs

Configure the two CTAs that appear in your footer:

```env
# Primary Footer CTA
VITE_CTA_FOOTER_PRIMARY_HEADING="Start Your Content Journey"
VITE_CTA_FOOTER_PRIMARY_SUBHEADING="Download AI CMS and begin creating professional content today"
VITE_CTA_FOOTER_PRIMARY_BUTTON_TEXT="Download Now"
VITE_CTA_FOOTER_PRIMARY_BUTTON_URL="https://github.com/yourusername/ai-cms"
VITE_CTA_FOOTER_PRIMARY_ENABLED="true"

# Secondary Footer CTA
VITE_CTA_FOOTER_SECONDARY_HEADING="Explore the Documentation"
VITE_CTA_FOOTER_SECONDARY_SUBHEADING="Learn how to customize and deploy your AI CMS"
VITE_CTA_FOOTER_SECONDARY_BUTTON_TEXT="View Docs"
VITE_CTA_FOOTER_SECONDARY_BUTTON_URL="/resources/guides/installation-and-setup"
VITE_CTA_FOOTER_SECONDARY_ENABLED="true"
```

<Callout type="info" title="URL Handling">
  URLs starting with "http" will open in a new tab, while URLs starting with "/" will navigate within your site. This makes it easy to mix external and internal links.
</Callout>

### CTA Variants

The `<CTASection>` component supports different visual styles:

- **`default`**: Standard CTA with rounded corners and padding
- **`compact`**: Smaller version for sidebars or tight spaces
- **`hero`**: Large, prominent CTA for landing pages
- **`footer`**: Optimized for footer placement

### Adding Custom CTAs

You can add CTAs to any page using the component:

```tsx
import { CTASection } from '../components/CTASection';

// In your component
<CTASection
  ctaName="my-custom-cta"
  variant="default"
  className="my-4"
  onButtonClick={(ctaName, buttonUrl) => {
    // Optional click tracking
    console.log(`CTA clicked: ${ctaName}`);
  }}
/>
```

Then configure it with environment variables:

```env
VITE_CTA_MY_CUSTOM_CTA_HEADING="Your Custom Heading"
VITE_CTA_MY_CUSTOM_CTA_SUBHEADING="Your compelling subheading"
VITE_CTA_MY_CUSTOM_CTA_BUTTON_TEXT="Action Button"
VITE_CTA_MY_CUSTOM_CTA_BUTTON_URL="https://example.com"
VITE_CTA_MY_CUSTOM_CTA_ENABLED="true"
```

### Disabling CTAs

To hide any CTA completely, set its `ENABLED` value to `false`:

```env
VITE_CTA_BLOG_POST_ENABLED="false"
```

**Writing Effective CTAs:**

**Technical Considerations:**
